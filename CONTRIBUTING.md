# Contributing

Contributions are welcome. Please keep additions factual and easy to verify.

## Add a Model or System

Include:

- system name;
- project page, repository, model card, documentation, or paper;
- year;
- system family: F1, F2, F3, F4, or F5;
- native input;
- output contract;
- access mode or parameter scale when public.

## Add a Benchmark

Include:

- benchmark name;
- task layer;
- input unit;
- expected output;
- main metric;
- dataset scale when public;
- official website, paper, or repository.

## Style

- Prefer stable official links.
- Avoid ranking claims unless backed by a benchmark citation.
- Use numeric metrics for evaluation summaries.
- Keep commercial systems separate from open models when reporting access mode.
